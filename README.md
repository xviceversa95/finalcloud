Аутентификация пользователей реализована на базе JWT-токенов и включает следующие этапы:

1. Вход (login).Пользователь отправляет логин и пароль. Система проверяет их через AuthenticationManager. Если данные верны, создаётся JWT-токен с именем пользователя, сроком действия и подписью. Токен возвращается клиенту.
   В контекст безопасности устанавливается аутентификация. 
2. Проверка токена.При каждом запросе токен проверяется: подтверждается подпись, срок действия и отсутствие в чёрном списке.
3. Загрузка пользователя. Для проверки используется MyUserDetailsService, который загружает пользователя из базы по имени.
4. Выход (logout).Для выхода пользователь передаёт auth-token. Сервер извлекает JWT и добавляет его в чёрный список, чтобы токен больше не принимался.


Основные компоненты
- UserController — обрабатывает запросы на вход и выход.
- AuthService — проверяет учётные данные и создаёт токен.
- JWTService — генерирует, проверяет и извлекает данные из JWT.
- MyUserDetailsService — загружает пользователя из базы.

Работа с файлами: 
1. Загрузка файла.
Клиент отправляет файл и его имя вместе с auth-token.
Сервер проверяет валидность токена и права пользователя.
Файл сохраняется на диске с уникальным именем, а метаданные (имя, тип, путь, владелец и т.д.) записываются в базу данных.
3. Скачивание файла.
Клиент указывает имя файла и передаёт auth-token.
Сервер проверяет токен, находит файл по имени и владельцу, возвращает его с корректными заголовками для скачивания.
4. Удаление файла.
Клиент указывает имя файла и передаёт auth-token.
Сервер проверяет права, удаляет файл с диска, если удалено - и соответствующую запись из базы.
5. Переименование файла.
Клиент отправляет запрос, в котором старое и новое имя файла, а также auth-token.
Сервер проверяет права, обновляет имя файла в базе данных.
6. Получение списка файлов.
Клиент передаёт auth-token и лимит.
Сервер возвращает список файлов, принадлежащих текущему пользователю.
7. Обработка ошибок.
Обрабатываются глобально (Advice Controller)   

Основные компоненты
- FileController — обрабатывает HTTP-запросы к файлам.
- FileService — реализует бизнес-логику: загрузка, скачивание, удаление, переименование, получение списка.
- FileRepository — работает с базой данных (MongoDB), хранит метаданные файлов.
- FileMapper — преобразует метаданные в DTO для передачи клиенту.
GlobalFileExceptionHandler — централизованно обрабатывает ошибки.
